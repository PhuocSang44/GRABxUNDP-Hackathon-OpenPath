# Core Beliefs

- The repository is the system of record for agents. Chat summaries are not.
- `AGENTS.md` is a router, not an encyclopedia — keep it short.
- Verification evidence matters more than confidence. "I think it works" is not evidence.
- One bounded task is better than many half-finished tasks. Finish feat-N before starting feat-N+1.
- Repeated human feedback should become a reusable harness rule, linter, or automated check — not a repeated chat reminder.
- Cleanup and simplification are part of shipping, not afterthoughts.
- If an agent cannot discover a fact in-repo, treat that fact as operationally unavailable.
- Accessibility score is a trust signal — never fabricate or smooth a score without recording source and confidence.
- The map is the product. Every feature must be visible and usable on the map before it is considered done.
